# HEARTBEAT.md - CMO主动检查清单

## 核心目标
驱动私域成交，完成公司战略目标（2026年营收¥615,000+，30单R90）

## 5个主动检查时段

### 06:00｜晨间排期检查
**任务：**
- 检查今日朋友圈排期是否已确认
- 如未确认，立即推送 A/B/C 三版文案 + 配图建议
- 目标：确保9:00前老徐确认全天计划

**检查清单：**
- [ ] 今日朋友圈几条内容？
- [ ] 每条是否有A/B/C三版文案？
- [ ] 配图建议是否已提供？
- [ ] 老徐是否已确认？

---

### 11:00｜上午内容执行检查
**任务：**
- 检查上午朋友圈是否已发布
- 如未发布，主动提醒并询问是否需要调整
- 收集已发布内容的数据反馈

**检查清单：**
- [ ] 上午内容是否已发布？
- [ ] 如未发布，询问原因并提供支持
- [ ] 记录发布时间、内容主题

---

### 14:00｜下午内容执行检查
**任务：**
- 检查下午朋友圈是否已发布
- 监控今日私域互动情况（询盘、评论、私信）
- 如有询盘，提醒老徐及时跟进

**检查清单：**
- [ ] 下午内容是否已发布？
- [ ] 今日是否有询盘/私信？
- [ ] 是否需要跟进提醒？

---

### 19:00｜晚间内容执行检查
**任务：**
- 检查晚间朋友圈是否已发布
- 汇总今日所有发布内容
- 准备数据复盘所需材料

**检查清单：**
- [ ] 晚间内容是否已发布？
- [ ] 今日共发布几条？
- [ ] 准备复盘数据

---

### 22:00｜数据复盘与次日规划
**任务：**
- 提醒老徐进行全天数据复盘
- 推送飞书多维表格（内容发布数据复盘表）
- 索要关键数据（阅读量、互动量、询盘数、成交线索）
- 准备次日排期初稿

**检查清单：**
- [ ] 复盘提醒是否已发送？
- [ ] 数据表格是否已推送？
- [ ] 关键数据是否已收集？
- [ ] 次日排期是否已准备？

---

## 深度内容触发机制

**当以下情况发生时，主动触发深度内容开发：**
1. 老徐提到超过2条以上内容想法
2. 某个话题讨论超过10分钟
3. 判断某话题适合深度长文（有痛点、有洞察、有案例）

**立即执行：**
- 主动询问细节，引导准备素材
- 关联多平台（朋友圈/小红书/视频号/公众号）
- 提供整体排期建议

## 反向Push机制

**当老徐未执行时，主动Push：**
- 超过2小时未确认排期 → "排期待确认，今日计划：___"
- 超过4小时未发布内容 → "今日内容还未发布，需要我调整吗？"
- 未回复数据复盘 → "昨日数据复盘还缺___，方便时补一下？"
- 有询盘未跟进 → "今日有X条询盘，建议___点前跟进"
