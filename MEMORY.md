## 🧠 知识库检索规则（2026-04-05 更新）

### 核心原则：先搜再写

在生产任何内容之前，必须先检索 Obsidian 知识库，查看是否有：
- 同主题的旧文章/案例/话术（可以复用）
- 已有的方法论/数据（可以引用）
- 相关的客户案例记录

### 知识源优先级
1. **Obsidian笔记库** → 最高权威（已沉淀的方法论、客户记录）
2. **飞书多维表格** → 朋友圈文案库（历史文案、数据复盘）
3. **Skills配置文件** → 运营级
4. **飞书文档** → 参考级

### 检索输出格式
每次检索返回三部分：
1. **摘要**（3行以内）
2. **引用**（具体出处）
3. **如有冲突信息，主动提示**

### 具体场景
- **写朋友圈前** → 先搜**飞书多维表格**「朋友圈文案库」同主题旧案例/话术（方案A：朋友圈仅存在多维表格，Obsidian 仅保留精选爆款）
- **写公众号前** → 先搜 Obsidian 知识库已有方法论/数据可引用
- **客户跟进前** → 先搜 Obsidian 客户历史记录
- **做选题时** → 先搜知识库里哪些主题还没写过

### 朋友圈存储策略（方案A：最小可行）
| 存储位置 | 内容 | 更新方式 |
|----------|------|----------|
| **飞书多维表格（主库）** | 所有朋友圈文案 + 数据 | 市场总监写入 |
| **Obsidian（精选库）** | 仅爆款（收藏率>5% 或 点赞>50）| 每月人工筛选同步 |

### 检索注意事项
- 只展示最相关的 **3条结果**，不要信息轰炸
- 标注每条结果的**更新时间**，优先用最新的
- 超过**6个月**没更新的内容降低优先级
- 归档：文件名后加 `（已归档）` 或 `（源文件·已升级归档）`
- 技能ID：英文小写+连字符，如 `moments-official`
