## 各系统分工

### 1. MEMORY.md — 长期战略记忆

**用途：**
- 公司战略方向
- 重要决策记录
- 品牌标准规范
- 关键客户信息
- 长期项目规划

**更新频率：**
- 每次重要决策后
- 每周回顾时
- 手动编辑

**示例内容：**
```markdown
## 公司北极星指标
- 2026年营收目标：¥615,000
- R90成交数量：≥30单

## 品牌声音标准
- 短句，最长15字
- 第一人称"我"
- 无AI味、无推销腔
```

---

### 2. Memory-OS — 自动语义记忆

**用途：**
- 对话内容自动保存
- 语义搜索检索
- 事实信息存储
- 跨会话关联

**特点：**
- 自动保存（无需手动）
- 语义搜索（自然语言查询）
- 本地存储（隐私安全）
- 免费使用（无API成本）

**使用示例：**
```bash
# 保存记忆
memory-os remember "老徐买了得宝纸巾18包"

# 语义搜索
memory-os search --semantic "纸巾购买"
```

---

### 3. .learnings/ — 学习改进记录

**用途：**
- 错误记录（ERRORS.md）
- 学习点记录（LEARNINGS.md）
- 功能请求（FEATURE_REQUESTS.md）
- 持续改进

**更新触发：**
- 命令失败时
- 用户纠正时
- 发现更好方法时

---

### 4. memory/YYYY-MM-DD.md — 每日归档

**用途：**
- 每日自动记录
- 会话历史摘要
- 决策时间线
- 审计追踪

**更新频率：**
- 每日自动创建
- 重要事件记录

## 使用流程

### 信息输入
```
新信息 → 判断类型 → 存入对应系统
            ↓
    战略/决策 → MEMORY.md
    对话/事实 → Memory-OS
    错误/改进 → .learnings/
    日常记录 → memory/YYYY-MM-DD.md
```

### 信息检索
```
查询需求 → 选择系统 → 获取信息
            ↓
    长期战略 → 读 MEMORY.md
    具体事实 → Memory-OS 搜索
    经验教训 → 读 .learnings/
    历史记录 → 读 memory/日期.md
```

## 记忆恢复流程（Bot 启动）

每个 Bot 启动时：

1. **读取 MEMORY.md** — 获取长期战略
2. **检查 Memory-OS** — 确认语义搜索可用
3. **读取 TODO.md** — 获取当前任务
4. **读取今日记忆** — 获取今日进展

```bash
# 自动执行
python3 ~/.openclaw/workspace/scripts/restore_bot_memory.py
```
