### 3.2 关键风险详细预案

**R01: 前20单成交不足**

```
触发条件：Week 2结束成交<10单

应对措施（按优先级）：
1. 价格测试：早鸟价从¥6,800降至¥4,800（限时1周）
2. 免费试用：提供3天免费体验，满意后付款
3. 案例置换：为3个种子客户免费服务，换取深度案例
4. 渠道拓展：启动合伙人分销，佣金提升至30%

决策人：老徐
决策时限：触发后48小时内
```

**R02: 客户投诉/退款**

```
触发条件：单周投诉≥2起 或 退款率>10%

应对措施：
1. 立即暂停新客签约，集中处理存量
2. 启动客户满意度调研，定位问题根源
3. 产品侧：优化交付流程，增加人工兜底
4. 销售侧：强化前置筛选，提高准入门槛
5. 财务侧：准备退款准备金（按20%计提）

决策人：产品经理+老徐
决策时限：触发后24小时内
```

**R03: 老徐时间超支**

```
触发条件：单周实际投入>计划20%

监控指标：
- 每周实际投入小时数
- 单客实际耗时vs计划耗时
- GTD系统任务完成率

应对措施：
1. 立即优化：识别耗时环节，启动优化
2. 资源补充：增加外包支持，减轻老徐负担
3. 流程迭代：更新SOP，固化效率提升措施
4. 客户管理：适当延长交付周期，降低密度

决策人：项目经理
决策时限：持续监控，周度复盘
```
