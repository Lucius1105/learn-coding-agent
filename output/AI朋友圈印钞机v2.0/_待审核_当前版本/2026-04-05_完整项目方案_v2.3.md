## 二、产品方案（产品经理）

### 2.1 产品定位

**核心价值主张：**
> 7天重构朋友圈变现系统，让客户主动找上门——不是发更多广告，而是让每条内容都成为自动成交的印钞机。

**4个差异化卖点：**
1. **真·AI原生工作流** — 不是教"用AI写文案"，而是搭建7天可跑通的自动化内容系统
2. **真·高客单价适配** — 专门针对"服务贵、决策慢"的行业
3. **真·7天见效承诺** — 带着业务进来，7天后带走能跑的系统
4. **真·老徐1v1诊断** — 商业教练+AI自动化双重加持

### 2.2 7天服务流程

| 天数 | 老徐动作 | 时间投入 | 客户交付物 |
|------|----------|----------|-----------|
| **Day 1** | 1v1语音信息采集（录音） | 1.5小时 | 录音文件转文字 |
| **Day 2** | 追问补充+AI整理 | 1小时 | 信息采集报告 |
| **Day 3** | 扣子系统配置 | 2小时 | 客户专属AI系统 |
| **Day 4** | 30天策略表制作 | 2小时 | 30天朋友圈内容策略表 |
| **Day 5** | 7条样本文案生成 | 1.5小时 | 7条定制文案样本 |
| **Day 6** | 飞书交付包整理 | 1小时 | 飞书5文件夹交付包 |
| **Day 7** | 交付说明会（30分钟视频） | 0.5小时 | 系统使用指南 |
| **售后** | 每周集中答疑1小时 | 0.5小时/周×4 | 社群答疑 |
| **总计** | — | **10小时/客户** | — |

### 2.3 信息采集问题清单（28问）

**Day 1 基础信息（14问）：**

1. 你现在的主营业务是什么？一句话描述。
2. 你的目标客户是谁？（年龄/性别/职业/收入水平）
3. 客户找你解决的核心问题是什么？
4. 客户为什么选择你，而不是竞争对手？
5. 你现在的客单价是多少？成交周期多长？
6. 你目前朋友圈发什么内容最多？（产品/生活/专业/其他）
7. 你发朋友圈的频率是怎样的？每天几条？
8. 你现在的朋友圈互动情况如何？（点赞/评论/私信咨询）
9. 你希望通过朋友圈达到什么目标？（获客/招商/建立信任/其他）
10. 你平时什么时段最方便发朋友圈？
11. 你有什么禁忌话题或不想提的内容吗？
12. 你个人的性格特点是什么？（专业型/亲和型/励志型/其他）
13. 你有什么独特的个人经历或故事？（创业故事/转折点/高光时刻）
14. 你的价值观是什么？（用3个关键词描述）

**Day 2 深度挖掘（14问）：**

15. 你印象最深的客户案例是什么？（具体故事）
16. 客户最常问你的3个问题是什么？
17. 你客户最痛的3个痛点是什么？
18. 你解决这些痛点的独特方法是什么？
19. 你行业里最常见的误区是什么？
20. 你个人的"绝活"或核心竞争力是什么？
21. 你平时和客户聊天时，最常分享的观点是什么？
22. 你有什么"反常识"的行业洞察？
23. 你个人的生活方式有什么特点？（爱好/习惯/日常）
24. 你希望通过朋友圈塑造什么样的个人形象？
25. 你有什么"金句"或口头禅？
26. 你最怕客户对你有什么误解？
27. 你最希望客户记住你什么？
28. 如果只能用一句话介绍自己，你会说什么？

### 2.4 交付物清单

**飞书5文件夹结构：**

```
📁 【AI朋友圈印钞机2.0】- 客户姓名
├── 📁 01_我的AI系统（扣子登录信息+使用指南）
├── 📁 02_30天策略表（按日期排列的内容规划）
├── 📁 03_文案样本库（7条定制文案+100条模板）
├── 📁 04_配图素材包（可商用图片+设计模板）
└── 📁 05_数据追踪表（发布记录+效果追踪）
```

**5大核心交付物：**

1. **扣子AI系统** — 已配置好的朋友圈文案官，输入主题即可生成文案
2. **30天策略表** — 按日期排列的内容规划，每天发什么一目了然
3. **7条定制文案** — 基于信息采集生成的专属文案样本
4. **100条模板库** — 按行业分类的高转化文案模板
5. **数据追踪表** — 发布记录+效果追踪，持续优化

### 2.5 扣子系统架构

**3个核心工作流：**

1. **人设提取工作流** — 从信息采集报告中提取客户人设标签
2. **文案生成工作流** — 根据人设+主题生成定制文案
3. **内容审核工作流** — 检查文案合规性+优化建议

**客户指令集：**
- `/生成 主题` — 生成单条朋友圈文案
- `/批量 主题1,主题2,主题3` — 批量生成多条文案
- `/优化 文案内容` — 优化已有文案
- `/灵感` — 获取选题灵感

## 一、战略一致性检查

### 1. 产品定位是否清晰？

**评估：✅ 清晰，但需强化一句话定位**

**现状分析：**
- 产品定位明确：「用扣子搭建的 AI 自动化系统，帮高客单价专业人士和门店老板把朋友圈变成 24 小时不打烊的成交机器」
- 核心价值主张清晰：7天交付、AI原生工作流、高客单价适配
- 差异化卖点突出：不是课程、不是代运营、是系统交付

**优化建议：**
```
优化后的一句话定位：
"7天帮你搭一套AI朋友圈印钞机——每天20分钟，让客户主动找上门。"

优化要点：
1. 加入"20分钟"这个具体数字，降低决策门槛
2. 强调"客户主动找上门"这个结果，而非过程
3. 口语化表达，更像老徐会说的话
```

**需补充的定位锚点：**
- 明确"不适合谁"：年营收<30万的小个体、没有变现产品的纯IP
- 强化"适合谁"：有产品/服务、客单价>3000元、朋友圈是主要获客渠道

### 2. 定价策略是否合理？

**评估：✅ 合理，但早鸟价需附加条件**

**现状分析：**
| 价格 | 定位 | 评估 |
|------|------|------|
| ¥6,800 早鸟价 | 种子用户验证 | ✅ 合理，筛选高意向 |
| ¥9,800 标准价 | 规模化销售 | ✅ 与价值匹配 |
| ¥8,800 过渡价 | 中间档位 | ⚠️ 必要性存疑，建议取消 |

**优化建议：**

**1. 早鸟价附加条件（防羊毛党）**
```
早鸟价 ¥6,800 需满足以下任一条件：
- 老徐私域老客户（有过付费记录）
- 限时48小时内付款
- 预付定金 ¥1,000 锁定名额
```

**2. 取消 ¥8,800 过渡价**
- 原因：增加决策复杂度，客户会纠结"等降价"
- 建议：只有两档价格，早鸟价售罄后直接跳到 ¥9,800

**3. 增加分期付款选项**
```
¥9,800 标准价支持：
- 一次性付款：¥9,800
- 分两期：首付 ¥5,000，7天后付 ¥4,800
- 分三期：首付 ¥3,500，第30天 ¥3,200，第60天 ¥3,300（总价¥10,000）
```

### 3. 目标客户是否精准？

**评估：✅ 精准，但需强化第一梯队聚焦**

**现状分析：**

| 梯队 | 客户类型 | 付费能力 | 转化难度 | 建议优先级 |
|------|---------|---------|---------|-----------|
| 第一梯队 | 律师/咨询师/疗愈师 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | **主攻** |
| 第二梯队 | 门店老板 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 次攻 |
| 第三梯队 | 小老板/创业者 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 暂缓 |

**优化建议：**

**1. 第一阶段只打第一梯队**
- 原因：第一梯队客单价高、决策快、转介绍能力强
- 执行：前20单早鸟只开放给第一梯队客户
- 话术："这期只服务高客单价专业人士，门店老板请等下一期"

**2. 第二梯队作为增长引擎**
- 时机：第一梯队50单后启动
- 策略：本地商会合作、同城KOL分销

**3. 第三梯队暂缓**
- 原因：价格敏感度高、执行能力弱、退款风险高
- 处理：等产品打磨成熟后，推出"轻量版"（¥3,800）再触达

### 4. 里程碑是否可达成？

**评估：⚠️ 激进但可达成，需调整节奏**

**现状分析：**
```
原里程碑（24周达成100单）：
Week 1-2: 预热造势
Week 3-6: 冷启动期（0→20单）—— 4周20单，每周5单
Week 7-10: 口碑裂变期（21→50单）—— 4周30单，每周7.5单
Week 11-14: 过渡优化
Week 15-20: 规模化冲刺（51→100单）—— 6周50单，每周8单
Week 21-24: 目标达成
```

**关键问题：**
1. 冷启动期4周20单过于乐观（需每周5单）
2. 过渡优化期4周无销售目标，时间浪费
3. 整体周期24周过长，市场窗口可能关闭

**优化建议（v2.2里程碑）：**

```
优化后里程碑（16周达成100单）：

Week 1-2: 预热造势 + 产品打磨
Week 3-8: 冷启动期（0→20单）—— 6周20单，每周3-4单 ✅ 更现实
Week 9-12: 口碑裂变期（21→50单）—— 4周30单，每周7-8单
Week 13-16: 规模化冲刺（51→100单）—— 4周50单，每周12单

关键调整：
1. 冷启动期延长到6周，降低每周压力
2. 取消"过渡优化"空窗期
3. 整体周期从24周压缩到16周
4. 每周目标更平滑：3单→4单→7单→12单
```

## 二、关键风险识别

### 1. 最大风险点是什么？

**🔴 风险一：扣子技术稳定性（概率：中 | 影响：高）**

**风险描述：**
- 扣子平台可能出现服务中断、API限制、功能变更
- 客户依赖扣子系统，一旦出问题直接影响交付

**预警信号：**
- 扣子官方公告功能调整
- 客户反馈系统响应慢/无法使用
- 连续3天系统异常

**🔴 风险二：老徐时间瓶颈（概率：高 | 影响：高）**

**风险描述：**
- 按方案每单需老徐投入10小时，20单即200小时
- 老徐还有其他业务（R90、急诊室、培训课），时间冲突
- 一旦老徐时间被占满，交付质量下降

**预警信号：**
- 老徐每周工作时间>60小时
- 连续2周无法按时交付
- 客户投诉响应慢

**🟡 风险三：客户执行意愿不足（概率：中 | 影响：中）**

**风险描述：**
- 系统交付后，客户不坚持使用
- "买了不用"导致口碑差、退款率高

**预警信号：**
- 客户7天内未发布第一条AI生成内容
- 连续7天无系统互动
- 退款申请>5%

| 措施 | 优先级 | 执行时间 |
|------|--------|---------|
| 立即培养1名交付助理（老徐口述，助理执行） | 🔴 高 | 第1-4周 |
| 每单老徐时间从10小时压缩到6小时 | 🔴 高 | 第5单后 |
| 建立标准化信息采集模板，减少1v1时间 | 🟡 中 | 第3单后 |

| 措施 | 优先级 | 执行时间 |
|------|--------|---------|
| 交付时强制要求客户当场发布第一条内容 | 🔴 高 | 每单交付 |
| 建立7天打卡机制，每日提醒 | 🟡 中 | 启动前 |

### 3. 应急预案是什么？

**预案A：扣子平台故障**

```
触发条件：扣子服务中断>24小时

应急措施：
1. 立即启动人工兜底方案
2. 老徐亲自为受影响客户手写7天内容
3. 赠送额外30天陪跑服务作为补偿
4. 48小时内修复或迁移至备用平台

客户沟通话术：
"系统临时维护，这7天的内容我亲自帮你写，额外送你30天陪跑。"
```

**预案B：老徐时间超载**

```
触发条件：连续2周无法按时交付 或 每周工作>60小时

应急措施：
1. 立即停止新单销售（暂停接单）
2. 已收定金客户：协商延期或全额退款
3. 紧急招募交付助理（从老客户中筛选）
4. 优化流程，压缩每单交付时间至6小时

恢复条件：
- 积压订单清零
- 交付助理培训完成
- 每周工作<50小时
```

**预案C：退款率过高**

```
触发条件：退款率>10% 或 连续3单退款

应急措施：
1. 立即复盘退款原因（产品/交付/期望落差）
2. 暂停销售，修复问题
3. 已交付客户：主动回访，防止连锁退款
4. 调整销售话术，降低过度承诺

恢复条件：
- 退款原因分析完成
- 问题修复验证
- 连续5单无退款
```

## 三、优化建议

### 1. 产品层面优化建议

**优化1：简化7天交付流程**

```
原流程（10小时/单）：
Day 1-2: 信息采集（2小时）
Day 3-4: 内容策略（3小时）
Day 5-6: 系统搭建（3小时）
Day 7: 交付培训（2小时）

优化后流程（6小时/单）：
Day 1: 信息采集（1小时语音）
Day 2: AI整理+老徐审核（0.5小时）
Day 3-4: 系统配置（2小时，助理执行，老徐审核）
Day 5: 内容生成+样本打磨（1.5小时）
Day 6: 飞书打包+培训视频（1小时）

关键变化：
1. 信息采集从2天压缩到1天
2. 系统搭建由助理执行，老徐只审核
3. 培训从直播改为录播视频
```

**优化2：增加"使用保证金"机制**

```
机制设计：
- 客户付款时额外收取¥500使用保证金
- 完成30天打卡（每周至少发5条）后全额返还
- 未完成打卡则保证金不退，转为老徐1v1咨询费

目的：
1. 筛选真正有执行力的客户
2. 提高客户使用率和满意度
3. 降低退款率
```

**优化3：建立产品版本迭代机制**

```
版本规划：
v1.0（当前）：基础版，7天交付，¥6,800-9,800
v2.0（第30单后）：Plus版，增加30天陪跑，¥12,800
v3.0（第60单后）：企业版，多账号管理，¥29,800

迭代节奏：
- 每30单收集反馈，迭代一次
- 老客户免费升级
- 新版本涨价，老版本下架
```

### 4. 财务层面优化建议

**优化1：现金流"三账户"管理**

```
收款后资金分配：
- 运营账户（50%）：用于外包、投放、日常支出
- 利润账户（30%）：老徐个人收入，不动用
- 风险储备金（20%）：退款准备金、应急资金

执行规则：
- 每单收款后立即分配
- 风险储备金达到¥50,000后，超额部分转入利润账户
- 运营账户低于¥20,000时，暂停投放
```

**优化2：建立"单客经济模型"动态监控**

```
每周追踪指标：
| 指标 | 目标值 | 预警线 |
|------|--------|--------|
| 单客毛利 | ≥¥3,500 | <¥2,000 |
| 获客成本 | ≤¥500 | >¥1,000 |
| 退款率 | ≤5% | >10% |
| 转介绍率 | ≥20% | <10% |

动态调整：
- 单客毛利<¥2,000：立即停止投放，优化成本
- 获客成本>¥1,000：暂停投放，优化素材
- 退款率>10%：暂停销售，复盘产品
```

**优化3：设置"财务安全线"**

```
红色预警线（立即停止）：
- 账上现金<¥30,000
- 连续2周亏损
- 退款率>15%

黄色预警线（谨慎操作）：
- 账上现金<¥50,000
- 单周亏损
- 退款率>8%

安全线（正常运营）：
- 账上现金>¥50,000
```
