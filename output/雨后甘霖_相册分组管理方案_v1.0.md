# 雨后甘霖 · 市场宣发相册分组管理方案

> 版本：v1.0
> 创建：2026-03-21
> 适用：iPhone / Mac / NAS 统一相册管理
> 目标：高效整理、快速调用、跨平台同步

---

## 一、核心设计原则

### 1. 按「内容生命周期」分组
不是按日期，而是按内容状态和使用场景

### 2. 三级分类体系
```
一级：内容类型（视频/图文/素材）
二级：使用场景（待发布/已发布/归档）
三级：平台适配（视频号/小红书/朋友圈/公众号）
```

### 3. 命名规范
```
[状态]_[平台]_[主题]_[日期]_[版本]

示例：
✅ 待发布_视频号_V-P01我是谁_20260321_v1.mp4
✅ 已发布_小红书_商业教练认知_20260315_v2.jpg
✅ 素材_封面_深蓝品牌色_20260301.png
```

## 四、跨平台同步方案

### 方案A：iCloud 照片同步（推荐）

```
iPhone 拍摄/保存 → iCloud 同步 → Mac 自动同步
```

**优点**：
- 苹果生态无缝同步
- 相册分组自动同步
- 实时更新

**设置**：
1. iPhone：设置 → Apple ID → iCloud → 照片 → 开启「iCloud 照片」
2. Mac：系统设置 → Apple ID → iCloud → 照片 → 开启

---

### 方案B：NAS 同步（进阶）

```
iPhone/Mac → NAS 相册库 → 备份 + 多端访问
```

**文件夹结构**：
```
NAS/雨后甘霖_内容资产/
├── 01_待发布/
├── 02_本周发布/
├── 03_已发布_视频号/
├── 04_已发布_小红书/
├── 05_已发布_朋友圈/
├── 06_素材库/
└── 07_归档/
```

**工具**：
- iPhone：DS file / Qfile（群晖/威联通）
- Mac：Finder 连接 SMB
- 自动同步：Synology Photos / QuMagie

## 五、使用工作流

### 每日工作流

```
早上 9:00
└── 打开「02_本周发布/今天」
    └── 直接发布当天内容

上午 11:00
└── 拍摄新内容 → 存入「01_待发布/拍摄中」

下午 14:00
└── 剪辑完成 → 移入「01_待发布/审核中」
    └── CC/老徐审阅 → 通过 → 移入「02_本周发布」

晚上 22:00
└── 发布完成 → 移入「03-05_已发布_对应平台」
    └── 截图数据 → 存入对应文件夹
```

### 每周工作流

```
周日晚上
└── 整理「01_待发布」
    └── 确认下周排期 → 移入「02_本周发布/周一至周日」

每月底
└── 整理「03-05_已发布」
    └── 数据表现好的 → 标记「爆款」
    └── 过时内容 → 移入「07_归档」
```

## 七、相册清理规则

### 自动清理（每月执行）

- [ ] 「01_待发布」超过30天的废弃草稿 → 移入「07_归档/废弃素材」
- [ ] 「02_本周发布」上周已发布内容 → 移入「03-05_已发布」
- [ ] 重复照片（相同内容）→ 保留最高质量版本
- [ ] 截图缓存 → 删除无用截图

### 手动清理（每季度）

- [ ] 「07_归档」超过1年的内容 → 备份到外部硬盘后删除
- [ ] 「06_素材库」未使用的素材 → 评估是否保留
- [ ] 检查 iCloud 存储空间 → 优化存储
