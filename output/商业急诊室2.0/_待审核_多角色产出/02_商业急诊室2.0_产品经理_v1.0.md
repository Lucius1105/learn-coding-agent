## 一、产品形态设计

### 1.1 服务流程设计（前/中/后期）

#### 前期：信息采集与AI预诊（T-3天至T-1天）

```
客户旅程：
询盘确认 → 支付定金 → 填写问卷 → AI预诊分析 → 生成诊断框架
   ↓          ↓          ↓           ↓              ↓
  即时      即时      15分钟      自动完成       自动完成
```

**流程节点说明：**

| 节点 | 动作 | 负责方 | 交付物 | 时效 |
|------|------|--------|--------|------|
| 询盘确认 | 客户表达意向，确认基本信息 | 销售系统 | 客户档案 | 即时 |
| 支付定金 | 支付¥199早鸟价或¥980标准价 | 支付系统 | 订单确认 | 即时 |
| 填写问卷 | 客户填写15题智能问卷 | 客户 | 问卷数据 | 15分钟 |
| AI预诊分析 | AI分析问卷，生成诊断要点 | AI系统 | 诊断框架 | 自动 |
| 生成诊断框架 | 输出3大卡点预判+初步建议 | AI系统 | 预诊报告 | 自动 |

**关键设计：**
- 问卷填写后，系统自动发送「诊断框架预览」给客户，建立期待
- 预诊报告同步给老徐，提前了解客户情况
- 系统自动预约诊断时间（客户自助选择可用时段）

#### 中期：深度诊断会话（T日，30分钟）

```
诊断流程：
开场破冰(3分钟) → 卡点确认(8分钟) → 深度诊断(15分钟) → 方案建议(4分钟)
```

**会话结构设计：**

| 阶段 | 时长 | 核心动作 | 老徐角色 | AI辅助 |
|------|------|---------|---------|--------|
| 开场破冰 | 3分钟 | 建立信任，确认预诊结果 | 主持人 | 显示客户画像 |
| 卡点确认 | 8分钟 | 验证AI预判的3大卡点 | 诊断师 | 实时记录要点 |
| 深度诊断 | 15分钟 | 追问根因，挖掘深层问题 | 教练 | 提示追问方向 |
| 方案建议 | 4分钟 | 给出破局方向和R90衔接 | 顾问 | 显示案例参考 |

**AI辅助功能：**
- 实时显示客户画像和预诊要点
- 会话中自动记录关键信息
- 根据对话内容提示追问方向
- 自动标记可用于报告的关键语句

#### 后期：交付与跟进（T+1小时至T+7天）

```
跟进流程：
报告生成(1小时内) → 报告交付(即时) → 7天行动启动 → 进度追踪 → 转化跟进
```

| 节点 | 动作 | 负责方 | 交付物 | 时效 |
|------|------|--------|--------|------|
| 报告生成 | AI生成诊断报告初稿，老徐审核 | AI+老徐 | 诊断报告PDF | 1小时内 |
| 报告交付 | 系统自动发送报告给客户 | 系统 | 邮件+微信 | 即时 |
| 7天行动启动 | 客户收到首日行动任务 | 系统 | 行动清单Day1 | T+1天 |
| 进度追踪 | 每日自动推送行动任务 | 系统 | 每日任务 | T+1至T+7 |
| 转化跟进 | 第3天评估R90意向 | 销售系统 | R90邀约 | T+3天 |

### 1.2 交付物规格

#### 核心交付物1：《商业诊断报告》

**规格参数：**
- 格式：PDF，A4，可打印
- 页数：8-12页
- 交付时效：诊断结束后1小时内

**报告结构：**

```
封面页
├── 客户姓名/企业名称
├── 诊断日期
├── 诊断师：徐沛霖
└── 雨后甘霖品牌标识

执行摘要（1页）
├── 3大卡点总结
├── 核心建议（3条）
└── 预期效果

诊断详情（4-6页）
├── 卡点1：详细分析 + 根因挖掘 + 解决方案
├── 卡点2：详细分析 + 根因挖掘 + 解决方案
├── 卡点3：详细分析 + 根因挖掘 + 解决方案

├── 即时行动（本周可执行）
├── 短期行动（1个月内）
├── 中期行动（1-3个月）
└── 资源需求清单

└── 下一步行动指引
```

**设计要点：**
- 视觉风格：专业但不冰冷，体现「懂AI的商业教练」定位
- 关键数据：用图表可视化呈现客户业务现状
- 个性化：报告开头包含会话中的关键金句/洞察

**规格参数：**
- 格式：微信小程序/飞书多维表格
- 交付方式：每日自动推送
- 互动形式：可打卡、可记录、可反馈

**7天行动设计：**

| 天数 | 主题 | 行动任务 | 预计时长 | 交付物 |
|------|------|---------|---------|--------|

- 客户完成后可打卡，系统自动记录

- 第3天和第7天，系统自动生成进度报告

**规格参数：**
- 格式：PDF，2-3页
- 内容：同类客户成功案例（脱敏处理）
- 匹配逻辑：根据客户行业/规模/卡点自动匹配

**案例结构：**
- 客户背景（行业、规模、阶段）
- 面临卡点（与客户相似的困境）
- 破局方案（关键行动和决策）
- 实施效果（数据化成果）
- 关键启示（可借鉴的经验）

## 二、核心功能模块

### 2.1 15题智能问卷设计

#### 问卷设计原则

1. **漏斗式结构**：从宏观到微观，从现状到痛点
2. **选择题为主**：降低填写门槛，便于AI分析
3. **逻辑跳转**：根据答案动态调整后续问题
4. **时间控制**：15分钟内完成

#### 问卷题目设计

**第一部分：基础画像（3题）**

| 题号 | 题目 | 类型 | 选项/说明 |
|------|------|------|----------|
| Q1 | 您的企业目前处于哪个阶段？ | 单选 | 初创期(0-1年)/成长期(1-3年)/成熟期(3年以上) |
| Q2 | 您的企业年营收规模是？ | 单选 | <100万/100-300万/300-500万/500-1000万/>1000万 |
| Q3 | 您的团队规模是？ | 单选 | 1人/2-5人/6-10人/11-30人/>30人 |

**第二部分：业务现状（4题）**

| 题号 | 题目 | 类型 | 选项/说明 |
|------|------|------|----------|
| Q4 | 您目前的主要获客渠道是？ | 多选 | 朋友圈/小红书/视频号/抖音/线下/转介绍/其他 |
| Q5 | 过去3个月，您的业务增长情况是？ | 单选 | 快速增长(>30%)/平稳增长(10-30%)/基本持平/下滑 |
| Q6 | 您每天投入业务的时间大约是？ | 单选 | <4小时/4-6小时/6-8小时/>8小时 |
| Q7 | 您目前最大的时间黑洞是？ | 单选 | 客户沟通/内容生产/团队管理/杂事处理/其他 |

**第三部分：核心卡点（5题）**

| 题号 | 题目 | 类型 | 选项/说明 |
|------|------|------|----------|
| Q8 | 如果用1句话描述您现在的困境，最贴切的是？ | 单选 | 有客户但转化难/有流量但获客难/有团队但管理难/有想法但执行难/其他 |
| Q9 | 过去3个月，您最焦虑的3件事是？ | 多选 | 收入不稳定/客户流失/团队不给力/没时间/方向不清/竞争加剧/其他 |
| Q10 | 您尝试过哪些解决方案？效果如何？ | 多选 | 学课程/请顾问/招团队/做投放/换赛道/没试过 |
| Q11 | 您认为问题的根源可能在哪里？ | 单选 | 能力不足/资源不够/方向错误/执行力差/不知道 |
| Q12 | 如果只能解决1个问题，您会选择？ | 单选 | 获客/转化/团队/产品/时间 |

**第四部分：期望与资源（3题）**

| 题号 | 题目 | 类型 | 选项/说明 |
|------|------|------|----------|
| Q13 | 您希望通过诊断获得什么？ | 多选 | 明确方向/具体方案/资源对接/执行督促/其他 |
| Q14 | 您每月可用于业务提升的预算是？ | 单选 | <1000元/1000-5000元/5000-1万/1-3万/>3万 |
| Q15 | 您愿意投入多长时间来执行建议？ | 单选 | 每天1小时/每天2小时/每天3小时+/无法保证 |

#### 问卷评分逻辑

**卡点识别算法：**

```
输入：15题答案
输出：3大卡点 + 卡点权重 + 初步建议方向

算法逻辑：
1. 根据Q8、Q9、Q12识别主要卡点类型（获客/转化/团队/产品/时间）
2. 根据Q5、Q7、Q11判断卡点严重程度
3. 根据Q1、Q2、Q3匹配行业/阶段特征
4. 输出3个优先级排序的卡点 + 每个卡点的初步建议方向
```

**评分维度：**
- 卡点类型：获客型/转化型/团队型/产品型/时间型
- 卡点严重程度：1-5分
- 可解决性评估：高/中/低
- R90转化潜力：高/中/低


### 2.3 7天行动清单设计

#### 清单结构

**每日任务卡片：**

```
┌─────────────────────────────────────┐
│  Day X · [主题]                      │
│                                     │
│  🎯 今日目标：                        │
│  [一句话描述今日要达成的结果]           │
│                                     │
│  ✅ 行动任务：                        │

│                                     │
│  [ 完成打卡 ]  [ 需要帮助 ]           │
└─────────────────────────────────────┘
```

#### 7天详细设计

**Day 1：现状盘点**
- 目标：清晰认知当前业务的真实状况
- 任务1：列出过去30天的收入来源和金额
- 任务2：列出每天时间分配的实际情况
- 任务3：找出3个最消耗精力但产出最低的事项
- 交付物：现状盘点表

**Day 2：目标澄清**
- 目标：明确未来3个月的核心目标
- 任务1：回顾诊断报告中的建议方向
- 任务2：设定1个SMART目标（具体、可衡量、可达成、相关、有时限）
- 任务3：写下达成这个目标对你的意义
- 交付物：目标卡

**Day 3：资源盘点**
- 目标：盘点可用资源，找到杠杆点
- 任务1：列出你拥有的5个核心资源（人脉/技能/资金/时间/信息等）
- 任务2：找出1个可以借力的外部资源
- 任务3：评估哪些资源可以放大使用
- 交付物：资源清单

**Day 4：方案设计**
- 目标：设计3个可能的破局方案
- 任务1：针对核心卡点， brainstorm 3个解决方案
- 任务2：每个方案列出需要的资源和预期效果
- 任务3：初步评估每个方案的可行性
- 交付物：方案草稿

**Day 5：方案选择**
- 目标：选择最优方案，制定执行计划
- 任务1：用决策矩阵对比3个方案
- 任务2：选择1个最优方案
- 任务3：将方案拆解为可执行的步骤
- 交付物：执行计划

**Day 6：障碍预判**
- 目标：预判风险，提前准备
- 任务1：列出执行过程中可能遇到的3个障碍
- 任务2：为每个障碍设计1个应对策略
- 任务3：找到1个可以支持你的人
- 交付物：风险预案

**Day 7：启动准备**
- 目标：准备就绪，下周启动
- 任务1：列出下周要执行的3个行动项
- 任务2：为每个行动项安排具体时间
- 任务3：写下对自己的承诺
- 交付物：启动清单

#### 跟进机制

**自动提醒规则：**
- 每日上午9:00推送当日任务
- 下午6:00未完成时提醒
- 晚上8:00未完成时二次提醒
- 次日早上复盘昨日完成情况

**进度追踪：**
- 客户可每日打卡记录完成状态
- 系统记录完成率，生成进度报告
- 第3天和第7天自动发送阶段性总结

**人工介入触发：**
- 连续2天未打卡 → 系统自动发送鼓励消息
- 连续3天未打卡 → 老徐或助理人工跟进
- 客户主动求助 → 24小时内响应

### 2.4 后续跟进机制

#### 跟进时间线

```
诊断当天(T)
├── 诊断结束后1小时内：发送诊断报告
└── 诊断结束后2小时内：发送感谢消息 + 7天行动预告

诊断后第1天(T+1)
├── 上午9:00：推送Day1行动任务
└── 晚上8:00：提醒完成打卡

诊断后第3天(T+3)
├── 上午9:00：推送Day3行动任务
├── 下午：评估R90转化意向
└── 晚上：发送R90免费咨询邀约（如符合条件）

诊断后第7天(T+7)
├── 上午9:00：推送Day7行动任务
├── 生成7天行动完成报告
└── 发送满意度调研问卷

诊断后第14天(T+14)
├── 发送两周进展跟进消息
└── 提供进一步支持

诊断后第30天(T+30)
├── 发送月度回顾消息
├── 邀请分享成果或反馈
└── 长期价值维护
```

#### R90转化跟进

**转化触发条件：**
- 客户在诊断中明确表达深度服务意向
- 客户在7天行动中表现积极（完成率≥70%）
- 客户在问卷中表明预算和时间投入意愿

**转化流程：**
1. T+3天：发送R90免费咨询邀约（15分钟）
2. 客户确认后：预约免费咨询时间
3. 免费咨询：深度沟通R90价值，确认匹配度
4. 跟进：根据沟通结果推进签约

**转化话术原则：**
- 不硬推，让价值自然呈现
- 基于诊断报告中的建议，说明R90如何帮助落地
- 给予专属优惠窗口期（诊断后7天内签约享早鸟价）
