### 1.2 关键自动化节点详解

#### 节点1：AI预诊引擎

**输入：** 15题问卷答案（JSON格式）

**处理流程：**
- 解析问卷数据（客户画像、业务现状、痛点）
- 调用大模型API进行AI分析
- 输出诊断框架（3大卡点、严重程度、R90转化潜力）

**输出：** 诊断框架（JSON格式）

**自动化触发：** 问卷提交后实时执行

**预期耗时：** < 30秒

---

#### 节点2：AI报告生成引擎

**输入：**
- 问卷数据（JSON）
- 会话转录文本（TXT）
- 老徐标记的关键信息（JSON）

**处理流程：**
- 数据预处理，提取关键洞察
- 调用大模型API生成报告各章节
- 组装Markdown报告
- Markdown转PDF

**输出：** 诊断报告（Markdown + PDF）

**自动化触发：** 诊断会话结束后自动执行

**预期耗时：** 5-10分钟（AI生成）+ 老徐审核10分钟

---

#### 节点3：7天行动清单自动化

**系统架构：**
- 任务调度器（Cron Job）：每日定时推送任务
- 任务内容生成器：基于诊断报告生成个性化任务
- 消息推送服务：微信小程序/企业微信/邮件
- 打卡记录系统：进度实时同步

**任务生成逻辑：**
- 基于诊断报告卡点，匹配对应主题任务
- AI个性化优化任务描述
- 每日自动推送，支持打卡记录

---

#### 节点4：智能跟进系统

**跟进规则引擎：**
- 基础提醒规则：诊断完成1小时后发报告、每日9点推送任务、未完成20点提醒
- R90转化跟进：Day3评估意向、Day7发送证书和优惠
- 人工介入规则：连续2天未打卡通知助理、求助请求24小时内响应

## 二、数字化方案设计

### 2.1 系统架构图

```
【客户端】
├── 微信小程序（7天行动清单）
├── 企业微信（客户沟通）
└── 邮件（备用通道）

【接入层】
├── API Gateway（统一接口网关）
└── 管理后台Web（老徐/助理使用）

【服务层】
├── 问卷服务
├── AI预诊引擎
├── 报告生成服务
├── 行动清单服务
├── 消息推送服务
└── 预约管理服务

【AI层】
├── 大模型API（通义千问/文心）
└── 语音识别（飞书妙记/通义）

【数据层】
├── 客户数据库
├── 问卷数据库
├── 报告存储
└── 日志记录

【集成层】
├── 飞书生态集成

└── 会议系统集成
```

### 2.2 核心系统模块设计

#### 模块1：问卷系统

**技术选型：**
- **MVP方案：** 腾讯问卷/金数据（SaaS，零开发）
- **V1.1方案：** 自建问卷系统（Vue.js + Node.js + MongoDB）

**功能需求：**
| 功能 | MVP | V1.1 |
|------|-----|------|
| 15题问卷展示 | ✅ | ✅ |
| 逻辑跳转 | ❌ | ✅ |
| 自动保存 | ✅ | ✅ |
| 数据提交 | ✅ | ✅ |
| 实时分析 | ❌ | ✅ |

---

#### 模块2：AI预诊引擎

**技术架构：**
- 输入层：问卷数据解析、数据清洗
- 处理层：Prompt工程、大模型API调用、结果解析
- 输出层：诊断框架生成、置信度评估
- 优化层：缓存机制、结果验证

**Prompt设计：**
- 角色：商业诊断专家
- 任务：基于问卷分析3个核心卡点
- 输出：卡点名称、描述、严重程度、根因、R90转化潜力

**API调用方案：**
- 阿里云百炼大模型API（通义千问）
- 单次调用约2000 tokens
- 成本：约¥0.04/次，月均100单¥4/月

---

#### 模块3：报告生成系统

**技术流程：**
1. 数据聚合（问卷+会话+标记）
2. AI内容生成（摘要、卡点详情、行动建议）
3. 模板渲染（Markdown组装）
4. PDF生成（HTML转换）

**技术选型：**
- Markdown转PDF：Pandoc + wkhtmltopdf
- 或：Python WeasyPrint库

---

#### 模块4：7天行动清单系统

**技术方案：**
- **MVP：** 飞书多维表格（零开发）
- **V1.1：** 微信小程序（需开发）

**核心功能：**
- 每日任务自动推送
- 打卡记录同步
- 进度可视化
- 未完成自动提醒

---

#### 模块5：消息推送系统

|---------|--------|---------|--------|

### 3.2 AI预诊引擎详细设计

**输入：** 15题问卷答案

**处理流程：**
1. 数据解析与清洗
2. 构建分析Prompt
3. 调用大模型API
4. 结果解析与结构化
5. 输出诊断框架

**输出示例：**
```json
{
  "pain_points": [
    {
      "id": 1,
      "name": "获客瓶颈",
      "description": "有产品但缺乏稳定获客渠道",
      "severity": 4,
      "root_cause": "过度依赖单一渠道，缺乏系统化获客策略",
      "category": "获客"
    }
  ],
  "r90_potential": "高",
  "confidence": 0.85
}
```

### 3.3 AI报告生成引擎详细设计

**输入：** 问卷数据 + 会话转录 + 老徐标记

**生成流程：**
1. 提取关键洞察
2. 生成执行摘要
3. 生成卡点详情（3个）
4. 生成行动建议
5. 组装Markdown
6. 转换PDF

**质量保障：**
- 老徐审核环节（10分钟）
- 关键信息高亮显示
- 模板化+个性化平衡
