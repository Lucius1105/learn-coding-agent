# 群晖写入速度慢诊断报告

## 问题现象
- Blackmagic Disk Speed Test 测试群晖写入速度仅 **9MB/s**
- 测试持续 2-3 分钟未完成
- 网络拓扑：书房 → 预埋网线 → 客厅主路由 → 兮克交换机 → 群晖916

## 网络拓扑图

```
书房电脑
   │
   ▼ (预埋网线?)
客厅主路由 (光猫旁)
   │
   ▼ (预埋网线?)
兮克 SKS3200-5E1X 交换机 (电视旁)
   │
   ▼ (2.5G网线)
群晖916 (电视旁)
```

## 关键疑点
**预埋网线很可能是 Cat5 (百兆) 而非 Cat5e/Cat6 (千兆+)**

---

## 诊断方案（从易到难）

### 方案A：软件快速判断（推荐先做）

**步骤1：查看网卡协商速度**

在书房电脑上执行：
```bash
# macOS
ifconfig | grep -A 5 "en0\|en1" | grep "media:"

# 或者看网络偏好设置
```
- 如果显示 `100baseTX` = 百兆网线（ culprit！）
- 如果显示 `1000baseT` = 千兆，问题在其他地方
- 如果显示 `2500baseT` = 2.5G，网线没问题

**步骤2：群晖端确认**

登录群晖 DSM → 控制面板 → 网络 → 网络界面：
- 看连接速度显示多少

---

### 方案B：物理排查（如果软件显示千兆但仍慢）

**方法1：看网线外皮标识**
- 找到书房网口面板和客厅网口面板
- 拆开面板，看网线外皮印刷：
  - `CAT5` = 百兆（100Mbps）→ 罪魁祸首
  - `CAT5e` = 千兆（1Gbps）
  - `CAT6/CAT6a` = 万兆（10Gbps）

**方法2：临时飞线测试（最准）**

买一根 **Cat6 成品网线**（3-5米）：
1. 书房电脑 → 直接飞线到客厅主路由
2. 测试群晖速度
   - 如果速度提升到 100MB/s+ → 确认预埋线是瓶颈
   - 如果还是 9MB/s → 问题在群晖或交换机

---

### 方案C：兮克交换机排查

登录兮克交换机管理界面（通常 http://192.168.x.x）：
- 查看端口速率显示
- 确认群晖连接的端口显示 2.5G 或 1G

---

## 预期结果对照

| 场景 | 预期速度 |
|------|---------|
| 百兆网线 (Cat5) | ~11MB/s (理论上限12.5MB/s) |
| 千兆网线 (Cat5e) | ~100MB/s |
| 2.5G 网线 (Cat6) | ~250MB/s |
| 群晖916 机械硬盘上限 | ~200MB/s |

**你的 9MB/s 非常接近百兆网线的理论上限 (12.5MB/s)**，强烈怀疑是 Cat5 预埋网线。

---

## 解决方案（如果确认是 Cat5）

### 短期方案：利用现有管线

**方法1：抽线换线**
- 如果预埋管够粗，尝试把 Cat5 抽出来，换 Cat6 进去
- 风险：线可能抽不动，或者管里有弯卡住

**方法2：光纤 + 光电转换器**
- 利用原有管道穿光纤
- 书房：光纤 → 光电转换器 → 电脑
- 客厅：光纤 → 光电转换器 → 交换机
- 成本：约 200-300 元，可跑 10G

**方法3：隐形光纤（沿踢脚线走）**
- 买透明光纤线，沿踢脚线/门框走
- 几乎看不见，可跑 10G
- 成本：约 100-200 元

### 长期方案：Mesh 无线回程

如果预埋线确实是百兆，且无法更换：
- 考虑用 WiFi6/7 无线回程
- 华硕路由器之间的无线回程可能比百兆网线更快
- 但群晖916 接在交换机上，这个瓶颈依然存在
