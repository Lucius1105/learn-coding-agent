## 预期速度分析

| 连接路径 | 预期速度 | 说明 |
|---------|---------|------|
| 群晖 ↔ 主路由 | **200-250MB/s** | 2.5G 满速！ |
| M4 ↔ 群晖 | **30-70MB/s** | 无线 Mesh 瓶颈 |
| M4 ↔ 互联网 | **取决于宽带** | 通常不受影响 |

### 关键理解

**M4 访问群晖的速度 = 无线 Mesh 回程速度**

无线 Mesh 速度取决于：
- 两个华硕路由之间的距离
- 中间障碍物（你说只有一个木门，很好）
- 华硕路由的 WiFi6 性能

**估算：**
- 如果无线 Mesh 能跑 **300-500Mbps** → M4 访问群晖 **30-50MB/s**
- 如果无线 Mesh 能跑 **800Mbps+** → M4 访问群晖 **70-100MB/s**

**结论：比现在的 9MB/s 快 3-10 倍，但跑不满群晖的 2.5G**

## 搬迁步骤（明早执行）

### Step 1：准备工作
- [ ] 确认群晖电源线长度够到入户门位置
- [ ] 确认兮克交换机电源线长度够
- [ ] 准备一根短网线（Cat6，1-2米）

### Step 2：关闭设备
- [ ] 正常关机群晖（DSM → 控制面板 → 关机）
- [ ] 拔掉兮克交换机电源
- [ ] 记录当前所有网线连接（拍照备用）

### Step 3：重新布线

**入户门位置（新布局）：**
```
光猫 LAN口 ──网线── 华硕主路由 WAN口

华硕主路由 LAN口 ──网线── 兮克交换机 任意口

兮克交换机 任意口 ──网线── 群晖916 网口
```

**书房位置（保持不变）：**
- 华硕路由B 保持原位
- 拔掉原来连接客厅的那根预埋网线
- 华硕路由B 现在只插电，**无线 Mesh 自动连接**

### Step 4：开机测试

1. **先开兮克交换机**（等指示灯正常）
2. **再开群晖**（等 DSM 启动完成）
3. **检查华硕 App 或 Web 界面**：
   - 确认 Mesh 显示 "无线回程"
   - 确认两个路由都在线

### Step 5：速度测试

**测试1：主路由旁边设备测群晖**
- 用笔记本或手机在客厅连 WiFi
- 访问群晖，测速
- **预期：100MB/s+**

**测试2：书房 M4 测群晖**
- Blackmagic Disk Speed Test
- **预期：30-70MB/s**（比现在的 9MB/s 快）

**测试3：书房 WiFi 测互联网**
- Speedtest 测外网速度
- 确认和之前差不多

## 可能遇到的问题

### 问题1：Mesh 连不上
**现象**：书房路由离线
**解决**：
- 把书房路由搬到更靠近客厅的位置
- 或者按华硕路由重置键，重新配对 Mesh

### 问题2：群晖 IP 变了
**现象**：找不到群晖
**解决**：
- 华硕路由管理界面看设备列表
- 或者用 Synology Assistant 搜索

### 问题3：速度还是慢
**现象**：M4 访问群晖还是 9MB/s
**排查**：
- 确认 M4 连的是书房路由的 5G WiFi
- 确认不是连了 2.4G
- 检查华硕 App 里 Mesh 回程速率显示
