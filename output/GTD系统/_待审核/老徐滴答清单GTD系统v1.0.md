## 一、系统设计哲学

### 1.1 为什么之前混乱？

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 40个项目 | 过度分类，认知超载 | 合并为4个核心项目 |
| 多方法论混用 | 四象限+8维度+GTD冲突 | 只用「战略重要性」一个标准 |
| 颜色逻辑混乱 | 既表紧急又表类型 | 颜色只表战略重要性 |
| 标签记不住 | 太多标签，无规律 | 自动识别关键词，无需记忆 |

### 1.2 新系统核心

```
┌─────────────────────────────────────────┐
│  只回答一个问题：                       │
│  「这个任务对2026战略有多重要？」        │
│                                         │
│  不重要 = 不做/延后/删除                │
│  重要   = 立即安排时间执行              │
└─────────────────────────────────────────┘
```

## 四、每日执行流程

### 4.1 早上 5分钟（启动）

```
Step 1: 看 🔴 本周燃烧
        → 确认今天必须完成的1-3个任务
        
Step 2: 选 🔵 战略推进  
        → 选1个长期任务推进
        
Step 3: 检查 ⚪ 日常维护
        → 能不做就不做，或批量处理
        
Step 4: 开始执行
        → 先 🔴 后 🔵，⚪ 尽量不做
```

### 4.2 白天执行（核心原则）

```
黄金法则：
- 🔴 任务必须今天完成，其他都可以延后
- 🔵 任务每天至少推进1个，保持长期节奏
- ⚪ 任务集中批量处理（如每周固定时间）
- 新想法 → 立即扔进 🟢 未来火种，不立即执行
```

### 4.3 晚上 2分钟（收尾）

```
Step 1: 完成任务打勾
        → 获得成就感
        
Step 2: 未完成 🔴 任务 → 移到明天第一优先
        
Step 3: 新想法 → 扔进 🟢 未来火种
        
Step 4: 记录明日 🔴 任务（3个以内）
```

### 4.4 周日 30分钟（规划）

```
Step 1: 清空 🟢 未来火种
        → 筛选可启动的 → 移到 🔴 或 🔵
        → 放弃过期的 → 删除
        
Step 2: 设定下周 🔴 本周燃烧（3个目标）
        → 对齐2026战略
        → 确保直接影响CCCV或个人北极星
        
Step 3: 确认 🔵 战略推进 进度
        → 长期项目本周推进计划
        
Step 4: 批量处理 ⚪ 日常维护
        → 报税、报表等集中处理
        
Step 5: 周复盘（看颜色分布）
        → 🔴+🔵 应该占80%+
        → ⚪ 超过50% = 警告，需调整
```
