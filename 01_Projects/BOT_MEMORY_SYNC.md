# Bot 记忆同步方案

> 解决飞书 8 Bot 重置后失忆问题
> 创建日期：2026-03-30

---

## 问题

- 8 个 Bot（战略顾问/市场总监/销售总监等）各自独立会话
- 重置后（/new 或 HTTP 400）会话历史清空，Bot 失忆
- 需要统一记忆恢复机制

---

## 解决方案

### 第一层：启动时强制读取记忆文件

每个 Bot 会话启动时，自动执行：

```
1. 读取 MEMORY.md（长期记忆）
2. 读取 memory/YYYY-MM-DD.md（今日记忆）
3. 读取 01_Projects/TODO.md（当前任务）
4. 读取相关项目文件
```

### 第二层：Memory-OS 作为统一后端

- 所有 Bot 共享 ~/.memory-os/ 存储
- 关键决策自动保存到 Memory-OS
- 语义搜索跨 Bot 可用

### 第三层：飞书多维表格归档

- 重要决策同步到飞书表格
- 长期可追溯
- 人工检查点

---

## 实施步骤

### 步骤 1：修改 Bot 启动流程

在每个 Bot 的 SKILL.md 中添加：

```markdown
## 启动记忆恢复

会话开始时，自动执行：
1. 读取 ~/.openclaw/workspace/MEMORY.md
2. 读取 ~/.openclaw/workspace/memory/YYYY-MM-DD.md
3. 读取 ~/.openclaw/workspace/01_Projects/TODO.md
4. 报告已恢复的记忆摘要
```

### 步骤 2：关键操作自动保存

Bot 执行重要操作时：
- 保存到 Memory-OS
- 更新 MEMORY.md
- 记录到 .learnings/

### 步骤 3：定期记忆同步

每 30 分钟或关键决策后：
- 同步到飞书多维表格
- 备份记忆文件

---

## 记忆优先级

```
高优先级（必须恢复）：
- 当前活跃项目
- 今日待办事项
- 关键决策记录

中优先级（按需恢复）：
- 历史项目档案
- 学习记录
- 技能配置

低优先级（后台加载）：
- 已完成项目
- 旧日志
```
